# Dart Programming Practicum Report – Prime Number Checker
This document is a practicum report for a Prime Number Checker application developed using Dart. The project demonstrates the implementation of control flows (branching and looping) and structured Git workflows.

## Setup Process
---
### 1. Repository Preparation
* Created a new repository on GitHub.
* Cloned the repository to the local environment.
* Initialized the Dart project structure.

### 2. Git Branching Strategy
The project was developed using a feature-based branching strategy:
* **Branch `input-validation`**: Implemented logic to handle invalid or negative inputs.
* **Branch `prime-logic`**: Developed the core algorithm to identify prime numbers using loops.
* **Branch `history-feature`**: Added functionality to store and display check history.

---
## Implementation
### Main Features
* **Input Validation**: Uses `if` statements to ensure only positive integers are processed.
* **Prime Detection**: Employs a `for` loop to check divisors and determine primality.
* **Result History**: Utilizes a `List` and `for-in` loop to display all previous check results.

**Result:**
The features were successfully developed in their respective branches and merged into the `main` branch. The final program executes all test cases accurately.

## Result Analysis
* The program correctly identifies prime numbers (e.g., 2, 7) and non-prime numbers (e.g., 1, 10).
* Branching allows for clean error handling when users input zero or negative numbers.
* The Git history shows a clear progression of feature development through sequential commits.
